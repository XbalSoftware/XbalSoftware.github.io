EYEbrary User Manual

EYEbrary helps you create clear, professional patient reports using reusable clinical content. Build your own library, assemble reports quickly, and export polished PDFs.

New Report

Building a report in New Report

Workflow

  • Select entries from the sidebar
  • They appear in the report area
  • Edit as needed
  • Export as PDF

Custom entries

  • Tap the + button (top right)
  • Create a custom entry by free-typing

Editing content

  • Tap an entry to edit
  • Tap Expand Content for full-screen editing
Expanded content editing
Expanded mode highlights the active entry and makes long edits much easier.
---

Collapsed vs Expanded View

Collapsed entries view
  • Use collapse to simplify the view
  • Use expand when actively editing
---

Library

Purpose

  • Create and manage reusable entries
  • Organize by category
  • Build your long-term knowledge base

Features

  • Search entries instantly
  • Sort based on your workflow
  • Mark favorites
  • Edit or delete entries
Favorites are ideal for commonly used conditions or instructions.
---

Batch Select

  • Select multiple entries at once
  • Apply bulk actions:
    • Delete
    • Export
    • Recategorize
Batch Select is the fastest way to clean up or reorganize a large library.
---

Categories

  • Used to organize your entries
  • Appear in dropdown selectors (not chips)
  • Fully customizable
---

Libraries

  • Multiple libraries supported
  • Switch instantly between them
  • Useful for:
    • Different clinics
    • Specialty content
    • Testing vs production libraries
---

Importing Libraries

  • Merge → adds or updates entries
  • Replace → completely overwrites library
  • New Library → creates a separate library
Turn OFF “Normalize text on import” if your formatting is already correct.
---

Settings

  • Manage libraries
  • Edit categories
  • Import letterhead
  • Reset app
---

History

  • View recent reports
  • Restore previous work
  • No patient data is stored
---

Privacy

  • Patient names are optional
  • Patient names are NOT saved
  • History contains no identifying information
  • Draft restoration excludes patient names
---

Tips

  • Use Expand Content for long edits
  • Use Batch Select for bulk actions
  • Use ⌘Z / ⇧⌘Z for undo/redo
  • Organize categories early — it pays off later
---
